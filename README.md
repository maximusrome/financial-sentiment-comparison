# Financial Sentiment Classification: Domain-Specific vs General-Purpose NLP

**CS 4120 Final Project ‚Äî Spring 2026**
**Team:** Gavin Girard, Avi Srinivasan, Noah McCray, Max Rome

## Research question

How do domain-specific NLP approaches compare to general-purpose ones for financial sentiment classification?

1. Does domain-specific pretraining (FinBERT) outperform a general-purpose equivalent (BERT-base) fine-tuned on the same data?
2. How does classification performance vary across annotator-agreement tiers?

## Dataset

**Financial PhraseBank** (Malo et al., 2014) ‚Äî 4,846 financial news sentences labeled positive, negative, or neutral by 16 financial experts, with each sentence tagged by annotator-agreement tier (100%, 75‚Äì99%, 66‚Äì74%, 50‚Äì65%). Loaded via HuggingFace: `takala/financial_phrasebank`. Some models additionally use the **Loughran-McDonald** financial sentiment lexicon (Loughran & McDonald, 2011).

## Models

1. **TF-IDF + Logistic Regression** ‚Äî classical baseline.
2. **TF-IDF + Loughran-McDonald** ‚Äî baseline augmented with financial-lexicon features.
3. **Fine-tuned BERT** ‚Äî `bert-base-uncased`.
4. **Fine-tuned FinBERT** ‚Äî `ProsusAI/finbert`. The (3)‚Üí(4) comparison isolates the effect of domain-specific pretraining.

## Experimental protocol

All four models share a single train/val/test split (80/10/10, stratified by sentiment √ó agreement tier, seed 42), generated by `data_loader.py` and saved to `data/splits.parquet`.

Training uses only the train subset; hyperparameters are selected on val; every model is evaluated once on test. For the agreement-tier analysis, each model is trained on the full training set and evaluated on test subsets partitioned by agreement tier ‚Äî this holds training data constant so the tier effect is purely about test-example difficulty.

Primary metric: macro-F1 (the dataset is ~59% neutral, so plain accuracy is misleading). We additionally report accuracy and per-class precision, recall, and F1.

## Prediction output format

Each model writes one CSV to `predictions/<model>_predictions.csv` with two columns:

| Column            | Type          | Description                                            |
| ----------------- | ------------- | ------------------------------------------------------ |
| `sentence_id`     | string        | Matches `predictions/test_metadata.csv`.               |
| `predicted_label` | string or int | One of `negative`/`neutral`/`positive` or `0`/`1`/`2`. |

Canonical filenames: `tfidf_logreg_predictions.csv` (Gavin), `tfidf_logreg_lm_predictions.csv` (Noah), `bert_finetuned_predictions.csv` (Avi), `finbert_finetuned_predictions.csv` (Max).
